# Contributing / Как внести вклад

Спасибо, что хотите помочь проекту! 🙌

## С чего начать

1. Форкните репозиторий и создайте ветку: `git checkout -b feature/my-change`.
2. Установите окружение разработчика:
   ```bash
   make dev        # или: pip install -e ".[dev,parse]" && pre-commit install
   ```
3. Внесите изменения, добавьте тесты.
4. Прогоните проверки:
   ```bash
   make lint
   make test
   ```
5. Откройте Pull Request с понятным описанием.

## Добавление заданий (не требует программирования)

Самый простой вклад — новые варианты. Создайте YAML-файл в
`data/variants/<subject>/<slug>.yaml` по формату из [`data/README.md`](data/README.md)
и проверьте загрузку: `python scripts/seed.py`.

Пожалуйста, используйте **оригинальные** или свободно распространяемые задания и
указывайте источник в поле `description`, если это уместно.

## Стиль кода

- Форматирование и линт — [ruff](https://github.com/astral-sh/ruff)
  (`make fmt`). Pre-commit прогоняет это автоматически.
- Пишите type hints и короткие докстроки для публичных функций.
- Бизнес-логику держите в `services/`, запросы к БД — в `repositories/`,
  хендлеры оставляйте тонкими.

## Коммиты

Желательно в стиле [Conventional Commits](https://www.conventionalcommits.org/):
`feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`.

## Сообщения об ошибках

Открывайте issue: что произошло, что ожидали, шаги воспроизведения, логи/трейс.
